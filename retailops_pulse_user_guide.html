01 Overview

RetailOps Pulse turns your service-management exhaust — Jira issues, ServiceNow incidents, problems and change requests — into two things: a heat wall that shows where issues cluster, and a service agent that answers any problem with a structured Root Cause → Fix Now → Mitigation response drawn from your SOP knowledge bank.

It runs in two modes, switched from the black masthead at the top right:

ModeWhat it doesData path
Offline archiveYou upload dumps; the app analyses them entirely in your browser. Nothing is sent anywhere.CSV / JSON files → local analysis
Real-timeConnector interfaces for Jira, ServiceNow, Confluence and SharePoint stream new records into a proactive alert feed; the heat wall and KPIs update live.Connectors → alert stream (simulated in this build)
PrivacyThe single HTML file has no backend and makes no network calls. Your dumps never leave the page — safe for client data in demos.

02 Getting started in 60 seconds

1
Open retailops_pulse_app.html in any modern browser (Chrome, Edge, Safari, Firefox). No install, no server.
2
Press Load demo data in the left panel. 1,000 Jira issues, 1,000 ServiceNow records and 16 SOPs load instantly.
3
Watch the KPI band fill and the heat wall render. The darkest cells are your most frequent issue clusters.
4
Tap a dark cell → review the records → press Ask agent about this pattern. The agent replies with RCA, fix steps and mitigations.

03 Offline mode — working with your own dumps

The Data sources panel accepts three files. Upload any or all; the heat wall and agent use whatever is loaded.

  • Jira dump (CSV) — an issue export from your retail project.
  • ServiceNow dump (CSV) — incidents, problems and change requests in one file.
  • Knowledge bank (JSON) — your SOP articles. This is what powers the agent's answers.

Column names are matched case-insensitively, and sensible fallbacks apply (see §07). If a row has no category, the app infers one from the summary text. Priorities like Critical / High / Low are normalised to P1–P4.

Good to knowLoading a new file replaces the previous dataset for that source — it does not append. Re-upload to refresh after a new export.

04 Reading the heat wall

Rows are issue categories, ordered hottest-first. Columns are switchable with the segment control:

  • By month — Jul 25 → Jun 26. Look for seasonal spikes (the demo data peaks in Nov–Dec festive trade).
  • By region — SA provinces plus National/Online. Reveals store-network or regional deployment problems.
  • By priority — P1–P4 mix per category. A category heavy in P1/P2 deserves a problem record even if its total is modest.

The All / Jira / ServiceNow control filters the source. Cell shading runs light → black with rising frequency; the Total column anchors each row. Every cell is keyboard-accessible (Tab + Enter).

Drill-down drawer

Tapping a cell opens the drawer with up to 40 underlying records: ID, summary, priority tag (P1 P2 P3 P4), status, region and date. The button at the bottom hands the whole pattern to the agent as a pre-formed question.

05 The service agent

The agent runs on local knowledge-bank retrieval: it tokenises your question, scores every SOP on keyword, title and root-cause overlap, and answers from the best match — fully deterministic, works offline, no API keys.

Ways to ask

  • Describe the symptom — "EFT declines spiking with code 91", "checkout creating duplicate orders", "promo price not applying at till".
  • Paste a record number — INC0010042, RET-1042, PRB0004012, CHG0007005. The agent looks the record up, shows its details, and routes to the right SOP.
  • From the heat wall — drill into a cell and use Ask agent about this pattern.
  • From an alert (real-time) — every alert card has Diagnose with agent.
  • Quick chips — the suggestion row above the input covers the five most common retail scenarios.

Anatomy of an answer

BlockContents
Root cause analysisThe most likely underlying cause, from the matched SOP.
Fix nowOrdered, actionable recovery steps for the on-call team.
Mitigate for the futurePreventive measures: alerting thresholds, process changes, automation.
Data signalLive stats from your loaded data: matching record count, open now, average resolve hours, regional hotspot, month-over-month trend.
Source lineSOP ID, service area and match confidence (high / medium).
If the agent can't matchIt will say so rather than guess, and suggest naming the service area (POS, EFT, checkout, stock sync, WRewards…) or pasting a record ID. Low-confidence matches are never dressed up as certainty.

06 Real-time mode

1
Switch to Real-time in the masthead. If nothing is loaded yet, the demo dataset auto-loads so the heat wall has a baseline.
2
In Connectors, enter an instance URL and token for Jira, ServiceNow, Confluence or SharePoint and press Connect. In this build the link is simulated (badge shows Linked · sim).
3
Proactive: the alert feed starts streaming new records (~every 8s). P1s get a red edge and a toast; P2s amber. Each alert also lands in the dataset, so the heat wall and KPIs tick up live. Pause anytime with the Stream toggle.
4
Reactive: press Diagnose with agent on any alert, or just type your question — the agent behaves exactly as in offline mode.

07 Data formats

Jira CSV

key,summary,category,priority,status,created,resolved,team,region,labels
RET-1000,WRewards linking failing in app with error ERR-397,WRewards Loyalty,P1,Resolved,2025-10-23 16:33,2025-10-23 22:01,Loyalty Platform,KwaZulu-Natal,wrewards-loyalty

Required: key (or id), summary, created. Everything else is optional with fallbacks.

ServiceNow CSV

number,type,short_description,category,priority,state,opened_at,resolved_at,assignment_group,region,related_ci
INC0010000,Incident,Store W373 on 4G failover — degraded POS response,Store Network & Connectivity,3,In Progress,2025-12-21 19:12,,Network Ops,Gauteng,Store-Router-10

type is one of Incident / Problem / Change Request. Priority accepts 1–4 or P1–P4.

Knowledge bank JSON

{
  "name": "Retail Services SOP Knowledge Bank",
  "sops": [{
    "id": "SOP-PAY-001",
    "title": "EFT declines or timeouts spiking",
    "area": "Payments & EFT",
    "keywords": ["eft", "decline", "code 91", "timeout"],
    "rca": "Response code 91 spikes indicate…",
    "fix": ["Step one…", "Step two…"],
    "mitigation": ["Preventive measure one…"]
  }]
}

keywords drive matching hardest (4 points each), then title and area words (2), then RCA words (0.4). Write keywords the way engineers actually describe the symptom.

08 Troubleshooting

SymptomLikely cause & fix
Heat wall stays empty after uploadDate column unreadable — use YYYY-MM-DD HH:MM or ISO 8601. Rows without a parseable created/opened date are skipped.
Everything lands in one categoryYour dump has no category column and summaries don't mention service areas. Add the column, or enrich summaries.
Agent says it can't matchSOP JSON not loaded, or the question uses vocabulary absent from your keywords. Extend the SOP keywords arrays.
"Could not read that JSON"File isn't valid JSON or lacks a sops array. Validate against the schema in §07.
Months look wrong on the heat wallThe month axis covers Jul 2025 – Jun 2026 (the demo window). Records outside it appear in totals but not in month cells — switch to region or priority view.
No alerts in real-time modeConnect at least one connector, and check the Stream toggle is on.

09 Production wiring (beyond this demo)

The real-time connectors are interfaces today; taking them live needs a thin backend because browser pages can't hold credentials or cross CORS boundaries safely:

  • Jira — REST v3 /search for backfill; webhooks (jira:issue_created) for push.
  • ServiceNow — Table API on incident/problem/change_request; Business Rules or Event Management to push new records.
  • Confluence / SharePoint — CQL search and Microsoft Graph /search to pull SOP pages into the knowledge index on a schedule.
  • Stream path — webhook receiver → queue → WebSocket/SSE to the page, replacing the in-browser simulator with the same alert-card contract.
  • Agent upgrade path — the local retriever stays as fallback; an LLM layer can compose over the same SOP matches for richer answers.
